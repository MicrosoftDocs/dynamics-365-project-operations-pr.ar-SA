# <a name="transaction-origins---link-actuals-to-their-source"></a>أصول المعاملات - ربط القيم الفعلية بمصدرها

_**ينطبق علي:** ‏‫Project Operations للسيناريوهات المستندة إلى مورد/غير مخزنة‬، ‏‫النشر الخفيف – التعامل مع الفواتير الأولية‬_

يتم إنشاء سجلات أصل المعاملة لربط القيم الفعلية بمصدرها، مثل إدخالات الوقت وإدخالات المصروفات وسجلات استخدام المواد وفواتير المشروع.

يوضح المثال التالي المعالجة النموذجية لإدخالات الوقت في دورة حياة مشروع Project Operations.

> ![معالجة إدخالات الوقت في Project Operations.](media/basic-guide-17.png)
 
1. يتسبب إرسال إدخال وقت في إنشاء سطرين من أسطر دفتر اليومية: أحدهما للتكلفة والأخر للمبيعات غير المفوترة.
2. تتسبب الموافقة النهائية على إدخال وقت في إنشاء قيمتين فعليتين: أحدهما للتكلفة والأخرى للمبيعات غير المفوترة.
3. وعندما يقوم المستخدم بإنشاء فاتورة مشروع، يتم إنشاء حركة بند الفاتورة باستخدام بيانات من القيمة الفعلية للمبيعات غير المفوترة.
4. عند تأكيد الفاتورة، يتم إنشاء قيمتين فعليتين جديدتين: إلغاء المبيعات غير المفوترة وقمية فعلية للمبيعات المفوترة.

يؤدي كل حدث في سير عمل المعالجة هذا إلى بدء إنشاء السجلات في كيان أصل المعاملة للمساعدة في إنشاء تتبع للعلاقات بين هذه السجلات التي تم إنشاؤها عبر إدخال الوقت وسطر دفتر اليومية والتفاصيل الفعلية وسطر الفاتورة.
